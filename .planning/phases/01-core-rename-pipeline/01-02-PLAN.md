## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ReplaceTransform — replace could introduce path-separator chars (/ or \) in output filename | mitigate | After all transforms, validate newFilename does not contain Path.GetInvalidFileNameChars(); if it does, emit FailedResult with "Computed filename contains invalid characters: [chars]" |
| T-02-02 | Tampering | RenameOrchestrator destPath — computed new path must stay in source directory | mitigate | Verify Path.GetDirectoryName(Path.GetFullPath(destPath)) == Path.GetFullPath(directoryPath) before any MoveFile call; reject with FailedResult if not equal |
| T-02-03 | Denial of Service | Unanchored regex on remove patterns — catastrophic backtracking on adversarial filenames | accept | Developer tool; pattern inputs are author-controlled; no network surface |
| T-02-04 | Information Disclosure | FileDiscovery enumerates all files in user-specified directory | accept | User specified the directory; enumeration is the expected behavior |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages added in this plan; only existing verified packages used |
</threat_model>
