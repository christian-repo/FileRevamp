## Phase Goal

**As a** user who renames a batch of files, **I want to** have the tool automatically resolve filename collisions before touching any file, **so that** no file in my batch is silently skipped or overwritten due to a naming conflict.

<objective>
Build the two-pass orchestrator (SAFE-01, SAFE-02): a Plan() pass that computes all source→destination pairs, resolves collisions via Windows auto-numbering (file(1).csv), and returns an immutable proposal list — followed by an Execute() pass that acts on the plan.

Purpose: The Phase 1 orchestrator is a lazy IEnumerable<RenameResult> that cannot satisfy SAFE-01's all-or-nothing semantics (MoveFile may have already been called by the time a later collision is detected). This plan replaces that stub with a correct two-pass design.
Output: RenameProposal record, CollisionResolver class, refactored RenameOrchestrator with Plan() + Execute(), and tests for all three.
</objective>
